# Markdown 品質ルール

このVaultのすべてのノートが守るべきMarkdown品質基準。

---

## 見出し構造

- 見出しレベルは論理的な順序で使用する（H1 → H2 → H3）
- H1はノート1つにつき1つのみ（ノートのタイトル）
- 見出しをスキップしない（H1の次にH3は禁止）

**良い例:**
```markdown
# タイトル
## セクション
### サブセクション
```

**悪い例:**
```markdown
# タイトル
### サブセクション（H2をスキップしている）
```

---

## Frontmatter

すべてのノートの冒頭にfrontmatterを記述する：

```yaml
---
tags: [カテゴリ/サブカテゴリ]
created: YYYY-MM-DD
status: draft | ongoing | active | published | archived | raw
---
```

**statusの値:**

| 値 | 意味 |
|---|---|
| `draft` | 下書き中 |
| `ongoing` | 進行中（リサーチなど） |
| `active` | アクティブ（プロジェクトなど） |
| `raw` | 生のアイデア（未整理） |
| `published` | 公開済み |
| `archived` | アーカイブ済み |

---

## Wikilink

- ノート間のリンクは必ず `[[ファイル名]]` 形式を使う
- 外部URLは通常のMarkdownリンク `[テキスト](URL)` を使う
- エイリアスは `[[ファイル名|表示テキスト]]` 形式

---

## タグ

- タグは `#カテゴリ/サブカテゴリ` 形式を使う
- frontmatterの `tags:` フィールドでリスト形式で管理する
- インラインタグ（本文中の `#タグ`）は補足的に使用可

**タグ例:**
```
#記事/下書き
#記事/公開済み
#アイデア
#リサーチ
#プロジェクト
#daily
#rules
```

---

## その他のスタイル

- リストには `-` を使う（`*` は避ける）
- コードブロックには言語指定を付ける（` ```markdown ` など）
- テーブルの列数は一貫させる
- 空行はセクション間に1行入れる（2行以上は避ける）
